# Aura v2.0.0 更新说明

本次版本重点围绕 **Agent 架构优化、执行效率、多轮任务稳定性和桌面页面性能** 做了一轮系统升级。Aura 正在从“能调用工具的桌面聊天应用”，进一步演进为一个更可靠、更可观测、更适合真实工程任务的本地 Agent 工作台。

## Agent 架构升级

- 统一默认 Agent 执行核心，减少旧执行路径与新执行路径之间的分叉，让任务调度、工具调用、错误恢复和最终输出走更稳定的一套链路。
- 优化任务路由系统，将请求动态归类到 `local-readonly`、`local-write`、`web-lookup`、`browser-interactive` 等能力层级，降低不必要的工具挂载和重复尝试。
- 改进能力选择与工具路由策略，让本地读写、Web 检索、MCP、Skills、Plugins、浏览器能力在同一个 Runtime 中更清晰地协同。
- 增强任务边界识别，默认聚焦最新用户请求，避免长对话中旧目标干扰当前轮执行。

## 执行效率与多轮思考优化

- 大幅优化长任务执行流程，支持更清晰的任务树、阶段状态、工具事件和执行证据沉淀。
- 引入更稳健的工作记忆与任务进度记录机制，减少多轮任务中重复搜索、重复读取、重复分析的问题。
- 优化上下文压缩策略：根据模型上下文窗口、工具 schema、输出预算和历史消息体量动态计算压缩阈值，让长对话更容易持续推进。
- 增强 Provider 重试与恢复链路，对流式中断、上下文过长、工具参数异常等情况给出更可恢复的执行路径。
- 增加完成状态校验，避免 Agent 在缺少执行证据时误报“已完成”，让代码修改、文件生成和资料整理类任务更可信。

## Web 检索与资料任务增强

- 优化 `web_search`、`web_fetch`、`web_research` 的统一检索 Runtime，让搜索、抓取、深度研究可以共享缓存、失败记忆和后端健康状态。
- 增加 Tavily、Brave、DuckDuckGo、HTTP Readability、Jina Reader、Lightpanda 等检索/抓取路径的调度与 fallback 能力。
- 针对失败站点、超时、限流、认证失败等情况加入冷却策略，减少同一轮任务中对不可用来源的重复请求。
- Web 工具事件现在会输出更紧凑的来源、证据、尝试后端和结果摘要，方便在桌面界面中追踪资料来源。

## 页面性能与本地持久化优化

- 会话消息改为更细粒度的 SQLite 加载策略，减少启动时一次性加载大量历史消息导致的页面压力。
- 工具事件详情支持压缩与按需读取，长日志、长网页内容、长 shell 输出不会直接撑爆消息列表。
- 日志查看器引入虚拟列表渲染，大量运行日志下滚动和筛选更流畅。
- 会话搜索下沉到 SQLite 查询，提升大量历史会话下的搜索效率。
- 增强消息版本、工具事件详情、工作记忆和 Agent Run 日志的本地持久化能力，方便复盘和排障。

## 本地工具与安全体验

- 改进文件编辑工具链，针对 `apply_patch`、精确编辑、多文件编辑和 shell 产生的文件变更提供更清晰的预览、校验和证据记录。
- 强化高风险操作审批：Shell、文件写入、桌面操作、外部路径访问、MCP/Plugin 变更等行为会进入更明确的权限分类。
- 优化外部路径和 Aura 能力目录的访问策略，降低误写 `~/.aura` Skills / Plugins / MCP 配置目录的风险。
- macOS Computer Use 工具继续保持按需挂载，用于截图、输入、快捷键、应用切换等显式桌面任务。

## 开发者体验

- 增补并整理 Runtime、Provider、工具路由、上下文压缩、执行证据、编辑工具、Web 检索等模块的测试覆盖。
- 优化 Tauri sidecar 构建流程，构建时校验 Node runtime 与目标平台架构，减少跨平台打包后的运行时错误。
- 新增中英文 README、MIT License 和第三方声明占位文件，方便项目开源传播和后续贡献协作。

## 升级建议

- 首次启动新版本后，建议进入设置页检查 Provider Profile、默认模型、Web 检索配置和 MCP Server 状态。
- 如果你有大量历史会话，本版本会更依赖 SQLite 的按需加载能力；首次打开旧会话时可能会触发一次消息加载。
- macOS 桌面自动化能力仍需要系统授予辅助功能、自动化或屏幕录制权限。
