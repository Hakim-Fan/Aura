# Aura v2.1.0 更新说明

- 新增 Multi-Agent 协作能力：复杂任务可以派发多个独立执行单元并行处理，适合分片检索、批量分析、独立实现和结果验证等场景。
- 完善 Multi-Agent 任务收口：每个协作任务都会记录自身类型、状态和结果，父任务结束前会汇总执行证据，避免任务已经结束但子任务仍显示运行中的问题。
- 引入 verification agent 验证流程：当任务涉及多个协作执行单元或复杂产物生成时，Aura 会在最终回答前要求独立验证，降低“模型说完成但产物未验证”的风险。
- 优化长任务流式稳定性：模型流式返回中断时会保留已有工具结果、文件产物和阶段进度，并基于最近 checkpoint 继续恢复，而不是整轮失败重来。
- 改进工具结果治理：大工具输出会优先作为产物和摘要进入上下文，减少把完整原文反复塞回模型导致的长任务卡顿和上下文污染。
- 优化写文件体验：`write_file` 现在会显示更清晰的流式写入进度，写入完成或中断后都会正确更新状态，避免工具卡在“执行中”。
- 升级执行摘要 UI：完成后的摘要会展示工具调用、Multi-Agent 协作类型、执行次数、成功/失败状态和生成产物，便于快速判断任务是否真正落地。
