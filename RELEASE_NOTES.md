# Aura v2.1.2 更新说明

本次更新重点优化 Windows 安装体验、Computer Use 工具挂载逻辑、窗口表现、Skills 外部目录管理，以及部分目录打开和日志读取的响应速度。

## 安装包与发布

- 修复 Windows 安装后桌面和开始菜单找不到 Aura 启动入口的问题。
- Windows 发布产物恢复为常规安装包，并补充 Windows ARM64 构建产物。
- macOS DMG 构建流程增加失败重试和诊断输出，降低 CI 中偶发 DMG 打包失败对发布的影响。

## Computer Use

- 移除设置里的“Computer Use 默认自动允许”入口，默认不再挂载系统浏览器和桌面操作工具。
- 在聊天框下方的工具浮层中新增内置 `Computer Use` 分组，默认关闭；用户打开后，仅当前会话挂载相关工具。
- `Computer Use` 目前仅在 macOS 上显示和启用；Windows 和 Linux 默认隐藏并不挂载相关工具。
- 浏览器/桌面操作审批卡片优化展示，避免 `system_browser_open` 等审批内容显示为空对象 `{}`。
- 工具分组标题后的“全部打开/全部关闭”改为开关，修复批量开关只影响最后一个工具的问题。

## 窗口体验

- Windows 主窗口启动时会按当前显示器可用区域动态计算尺寸并居中，同时设置最大尺寸，避免遮挡任务栏或在大屏上过度放大。
- macOS 主窗口保持原有较合适的默认尺寸。
- 设置窗口关闭后不再释放，下次打开保留窗口实例；MCP 编辑窗口和日志窗口关闭后仍会释放资源。

## Skills 与外部目录

- Skill 设置支持绑定多个外部目录。
- 已绑定外部目录会逐条显示，并在每条右上角提供删除图标，支持清除单个外部目录绑定。
- 清除外部目录时，会同步移除该目录下 Skills 的启用状态，但不会删除原目录文件。

## 性能与稳定性

- 打开 Skills、Plugins、MCP、日志、Lightpanda 等目录时，优先使用已缓存路径，不再在点击时触发完整 Aura 初始化和资源扫描。
- 日志窗口打开日志目录、读取日志列表和读取历史日志时，改为只解析日志目录，不再扫描 Skills/Plugins。
- Aura 默认 Skills/Plugins 初始化改为只补缺失文件，避免每次初始化都重复拷贝已存在的默认资源。
