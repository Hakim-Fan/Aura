# Aura 更新日志

## v1.1.1（2026-05-05）

### 长对话持久能力

- 单次对话支持 **20-30 轮以上**的连续交互，不再因上下文溢出被迫中断
- 支持携带 **50+ 个文件**的工程会话，上下文始终处于可用范围内
- 单次工具调用链（含大量代码输出）不再撑爆整个会话

### 上下文智能压缩

- 当会话接近上下文上限时，自动触发上下文压缩，**压缩过程对用户完全透明**
- 压缩后保留关键摘要和近期交互，不影响后续对话质量
- 长时间运行的后台会话也能自动恢复稳定，无需用户手动干预
- System Prompt 在上下文紧张时自动精简，释放更多空间给实际对话

### 手动压缩能力

- 新增手动压缩入口，用户可在任意时刻主动压缩当前会话上下文
- 适合需要长时间维持同一会话、或主动希望释放上下文空间的场景

### LLM 调用优化

- 上下文接近上限时大幅减少不必要的模型调用（最长等待时长缩短约 50%）
- 上下文过大时降低最大输出预算，配合压缩策略快速恢复可用状态
- 整体响应速度和稳定性显著提升，减少了长时间等待后失败的情况
