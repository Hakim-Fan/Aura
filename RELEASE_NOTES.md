# Aura v2.1.3 更新说明

本次更新重点补齐应用内更新链路，增强项目级指令能力，并继续优化 Skill 安装、设置状态和工作区文件操作体验。

## 应用内更新

- 接入 Tauri 标准 updater，支持在 Aura 内直接检查、下载、安装更新并重启应用。
- Release 流程会生成并上传 `latest.json` 更新清单，App 会优先走签名校验后的应用内更新。
- 保留 GitHub Release 检查作为兜底路径：当标准 updater 检查失败时，仍可引导用户打开下载页。
- 更新弹窗增加下载进度、安装中状态和错误提示，避免用户误以为点击后无响应。
- updater 公钥已固定写入应用代码，发布时只需要 GitHub Actions 使用匹配的 `TAURI_SIGNING_PRIVATE_KEY` 签名更新包。

## 项目级 AGENTS.md

- 新增项目内 `.aura/AGENTS.md` 指令能力，用于记录当前工作区的项目级开发约定和 Agent 行为要求。
- 创建或初始化工作区时会自动准备 `.aura/AGENTS.md` 文件，方便用户直接编辑项目指令。
- Agent 运行时会读取当前工作区的 `.aura/AGENTS.md`，并作为项目级开发指令注入上下文。
- `AGENTS.md` 指令会在不违反系统安全、权限策略和最新用户请求的前提下生效。
- 对过长的项目指令做长度限制和截断提示，避免单个项目文件占用过多上下文。

## Skill 安装策略

- `aura_install_skill` 默认先安装到当前工作区 `.aura/skills`，并为当前会话启用。
- 安装成功后，Aura 会询问是否同步到全局 `~/.aura/skills`；拒绝后只保留工作区安装。
- 优化 npx Skill 安装：优先在隔离环境中执行 npx，若未产出有效 Skill，再回退到命令中引用的 URL、GitHub 或 npm 包来源。
- 增强 Skill 安装相关测试，覆盖 workspace-first、全局同步审批和 npx fallback 场景。

## 设置与工作区体验

- 修复和优化设置页状态归一化，减少旧配置或异常类型导致的开关状态不一致。
- 默认发送快捷键调整为 `Enter` 发送，`Shift + Enter` 换行；仍可在设置中切换为 `⌘/Ctrl + Enter` 发送。
- 工作区文件浏览器支持双击文件、预览内容或图片时用系统默认应用打开。
- 优化工作区文件预览区域的交互提示和布局细节。

## 稳定性

- 更新检查增强版本号解析和 GitHub Atom feed 兜底，减少 GitHub API 异常时无法发现新版本的问题。
- 优化能力安装和执行策略提示，减少通过 shell 手工修改 Aura 能力目录带来的不一致风险。
