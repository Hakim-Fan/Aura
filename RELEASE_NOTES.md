# Aura v2.0.1 更新说明

- 优化长任务续跑体验：参考 Claude 的轻量恢复策略，任务重新继续时会更明确地带入最近的 `task_progress`，让 Agent 知道“已经完成什么、当前进行到哪一步、下一步应该做什么”。
- 强化续跑防重复机制：如果旧进度里已经完成环境检查、文件读取、Skill 调用或脚本探测，新的继续任务会优先复用这些证据，减少重复读取 Skill、重复查找 Python、重复分析文件结构等问题。
- 改进任务进度选择逻辑：当历史中同时存在“更先进的旧进度”和“失败重启产生的新计划”时，Aura 会保留更有价值的已完成步骤，避免模型因为一次失败重试把任务计划重置到开头。
- 优化工具失败后的继续执行：已修复“早期失败记录长期污染当前任务状态”的问题，后续成功的同类工具执行可以清除旧失败影响，降低任务被错误判定为失败的概率。
- 增强长时间 Shell 命令状态更新：`exec_command` 启动的后台命令完成后，会自动回写原工具事件的最终状态、退出码和输出，避免界面长期显示运行中或成功状态不准确。
- 改进 Runtime 上下文提示：续跑上下文现在会同时保留工具证据、任务进度、附件观察和已调用 Skill 信息，帮助 Agent 更像“接着做”而不是“重新开始”。
